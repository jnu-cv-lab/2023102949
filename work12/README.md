# Sinusoidal Position Encoding 与 RoPE 实现及对比实验报告
## 一、实验概述
本实验基于 NumPy 实现**正弦位置编码（Sinusoidal Position Encoding）**、二维向量旋转、高维旋转位置编码（RoPE），对比 `E+pos` 加法式位置注入与 RoPE 旋转式位置编码的差异，并通过数值实验验证 RoPE 的**相对位置编码**特性，分析两种方案的优劣。

## 二、背景原理
### 2.1 Transformer 引入位置编码的原因
Transformer 依赖自注意力机制建模全局依赖，**本身不具备时序/位置感知能力**，无法区分序列中不同位置的元素。因此必须额外引入位置信息，让模型学习词向量与位置的关联。

### 2.2 正弦位置编码（Sinusoidal Position Encoding）原理
采用正弦、余弦函数生成固定位置编码，公式如下：
$$
\begin{cases}
PE(pos, 2i) = \sin\left(pos \cdot \dfrac{1}{10000^{\frac{2i}{d_{model}}}}\right)\\
PE(pos, 2i+1) = \cos\left(pos \cdot \dfrac{1}{10000^{\frac{2i}{d_{model}}}}\right)
\end{cases}
$$
- $pos$：序列位置下标
- $d_{model}$：词向量维度
- 偶数维度使用 $\sin$，奇数维度使用 $\cos$，生成与词向量同维度的位置向量。

### 2.3 E+pos 方案缺陷
传统方案将**词嵌入向量 E** 与**位置编码 pos** 直接相加融合：$Emb = E + PE$。
该方式存在**内容与位置特征强耦合**问题：词语义特征和位置特征混合在同一向量空间，相互干扰；同时该编码方式偏向**绝对位置**，对长序列、相对位置依赖场景建模能力弱。

### 2.4 RoPE 旋转位置编码原理
RoPE 摒弃加法，通过**二维旋转变换**向向量注入位置信息：
1. 将高维向量按**两两分组**划分为若干二维子向量；
2. 对每组二维向量根据位置 $pos$ 计算旋转角度 $\theta$，执行旋转变换；
3. 旋转公式：
$$
\begin{cases}
x_1' = x_1\cos\theta - x_2\sin\theta\\
x_2' = x_1\sin\theta + x_2\cos\theta
\end{cases}
$$
RoPE 仅作用于注意力的 Query(Q) 和 Key(K)，其点积结果会**天然包含相对位置信息**。

import numpy as np

def sinusoidal_pos_encoding(max_len: int, d_model: int) -> np.ndarray:
    # 初始化位置编码矩阵
    pe = np.zeros((max_len, d_model))
    # 所有位置下标
    pos = np.arange(0, max_len)[:, np.newaxis]
    # 计算公式里的分母项
    div_term = np.exp(np.arange(0, d_model, 2) * (-np.log(10000.0) / d_model))
    # 偶数维度 sin，奇数维度 cos
    pe[:, 0::2] = np.sin(pos * div_term)
    pe[:, 1::2] = np.cos(pos * div_term)
    return pe

def rotate_2d(x1: float, x2: float, theta: float) -> tuple[float, float]:
    # 计算余弦、正弦
    cos_t = np.cos(theta)
    sin_t = np.sin(theta)
    # 二维旋转公式
    x1_rot = x1 * cos_t - x2 * sin_t
    x2_rot = x1 * sin_t + x2 * cos_t
    return x1_rot, x2_rot

def rope_encoding(x: np.ndarray, pos: int, d_model: int) -> np.ndarray:
    x_rot = x.copy()
    for i in range(0, d_model, 2):
        # 计算旋转角度
        theta = 1.0 / (10000 ** (2 * i / d_model))
        angle = pos * theta
        cos_t = np.cos(angle)
        sin_t = np.sin(angle)
        # 两两分组旋转
        x1 = x[i]
        x2 = x[i+1]
        x_rot[i] = x1 * cos_t - x2 * sin_t
        x_rot[i+1] = x1 * sin_t + x2 * cos_t
    return x_rot

# 对整个序列批量使用RoPE
def apply_rope_to_seq(seq: np.ndarray, d_model: int) -> np.ndarray:
    seq_len = seq.shape[0]
    rope_seq = np.zeros_like(seq)
    for pos in range(seq_len):
        rope_seq[pos] = rope_encoding(seq[pos], pos, d_model)
    return rope_seq

if __name__ == "__main__":

五、核心问题解答
5.1 为什么 Transformer 需要位置编码？
Transformer 自注意力是排列无关的，无法感知序列顺序与位置，必须通过位置编码注入时序信息，模型才能理解文本、序列的先后关系。
5.2 传统 Sinusoidal 位置编码如何生成？
基于位置下标和向量维度，交替使用 
sin
、
cos
 函数计算，偶数维取正弦、奇数维取余弦，生成固定维度的位置向量。
5.3 E+pos 存在内容与位置混合的问题？
词语义向量与位置向量直接相加，两类特征在同一维度空间叠加、互相干扰；编码偏向绝对位置，泛化性差，长序列表现不佳。
5.4 RoPE 与加法式编码的本质区别
RoPE 不使用加法，而是对向量做旋转变换，将位置信息通过几何变换嵌入向量本身，而非额外拼接 / 叠加特征。
5.5 RoPE 为何作用在 Q、K 上？
自注意力核心是 
Q⋅K 
T
 
 点积运算，将 RoPE 施加于 Q、K，可让位置信息直接融入注意力打分过程，高效建模位置关联。
5.6 RoPE 点积天然包含相对位置
旋转角度由绝对位置决定，两个向量旋转后的点积会抵消绝对位置，最终结果仅由 ** 位置差值（相对位置）** 决定。
5.7 RoPE 相比 E+pos 更巧妙的原因
解耦特征：通过旋转变换嵌入位置，语义特征与位置特征干扰更小；
天然相对位置：注意力打分只和相对位置相关，适配长序列、长文本场景；
无额外参数 / 向量：无需单独存储位置编码，轻量化且推理友好；
泛化能力强：可支持超出训练最大长度的序列外推，传统加法位置编码难以做到。
六、实验总结
成功实现正弦位置编码、二维旋转、高维 RoPE 三大核心模块，代码运行结果符合理论；
E+pos 是绝对位置 + 加法融合，存在特征耦合问题；RoPE 是几何旋转 + 相对位置编码，设计更合理；
数值实验证明：RoPE 注意力点积仅依赖相对位置，是其在大模型、长序列任务中广泛使用的核心原因；
RoPE 在特征解耦、长序列泛化、外推能力上全面优于传统加法式正弦位置编码。
